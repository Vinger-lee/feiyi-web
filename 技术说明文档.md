# 热河守艺人非遗网站技术说明文档

## 1. 项目概述

本项目是一个面向承德、热河地区非物质文化遗产展示与智能问答的前端网站。网站提供非遗首页展示、非遗百科、分类浏览、收藏、历史记录、项目详情弹窗、Coze 智能体对话、3D 非遗星球等功能。

项目当前采用纯前端静态部署方式，可以通过 GitHub Pages 对外访问。生产构建产物输出到 `dist/`，由 GitHub Actions 自动发布。

## 2. 技术栈

| 类型 | 技术 |
| --- | --- |
| 前端框架 | React 19 |
| 构建工具 | Vite 8 |
| 语言 | TypeScript 6 |
| 样式 | Tailwind CSS 3 + 自定义 CSS |
| 3D 渲染 | Three.js、@react-three/fiber、@react-three/drei |
| 动画 | Framer Motion、AnimeJS、CSS keyframes |
| 智能体 | @coze/web-sdk、Coze API |
| 本地存储 | localStorage |
| 部署 | GitHub Actions + GitHub Pages |
| 测试/验收 | TypeScript、ESLint、Vite build、Playwright 手动脚本 |

## 4. 核心功能

### 4.1 首页

首页是网站的主要视觉入口，包含：

- 沉浸式 Hero 首屏。
- WebGL 粒子与光带背景。
- 搜索框，可搜索非遗项目。
- 快速按钮：打开百科、智能对话、随机探索、分类入口等。
- 精选项目展示区。
- 短暂出现的“向下滑动”提示，几秒后自动淡出。
- 非遗项目跑马灯。
- 首页下方的 3D 非遗星球区域。

相关文件：

- `src/App.tsx`
- `src/index.css`

### 4.2 非遗百科

非遗百科展示全部非遗项目，支持：

- 项目卡片展示。
- 搜索。
- 点击打开详情弹窗。
- 收藏项目。
- 图片本地化加载。

数据来源为 `src/data/heritageData.ts` 中的 `heritageData`。

### 4.3 分类浏览

分类基于 `HeritageCategory` 类型，当前分类包括：

- `craft`：传统技艺
- `art`：传统美术
- `performance`：传统戏剧
- `music`：传统音乐
- `dance`：传统舞蹈
- `literature`：民间文学
- `medicine`：传统医药
- `folk`：民俗
- `sport`：传统体育

分类名称、图标、描述定义在 `src/types/index.ts` 的 `HERITAGE_CATEGORIES`。

### 4.4 项目详情弹窗

点击项目卡片、3D 星球卡片或精选项目后，会打开详情弹窗。

详情包含：

- 项目主图。
- 项目名称。
- 分类、地区、等级。
- 简介。
- 标签。
- 详细介绍。
- 历史渊源。
- 核心技艺或主要习俗。
- 代表性传承人。
- 收藏按钮。

详情页对缺失字段做了 fallback，避免 `details`、`masters`、图片字段缺失导致运行时崩溃。

### 4.5 3D 非遗星球

3D 星球是首页下方的交互展示区，使用 React Three Fiber 构建。

实现要点：

- `Canvas` 提供 WebGL 渲染环境。
- `GlobeStructure` 绘制球体、经纬线、粒子、外部轨道线。
- `createGlobeNodes` 使用黄金角分布将项目均匀分布在球面。
- `HeritageSurfaceCard` 将项目卡片绘制为 CanvasTexture，再映射到曲面网格上。
- `createCurvedCardGeometry` 生成符合球面曲率的卡片几何体，使卡片贴合球面。
- `OrbitControls` 支持拖拽旋转，并开启低速自动旋转。
- 根据相机方向判断卡片可见性，避免背面或边缘卡片穿出。
- 点击球面卡片会打开对应项目详情。

当前自转参数：

```tsx
autoRotate={!reducedMotion}
autoRotateSpeed={0.3}
```

如需调快或调慢星球速度，修改 `src/App.tsx` 中 `OrbitControls` 的 `autoRotateSpeed`。

### 4.6 智能对话

项目包含两种 Coze 相关接入：

- `src/App.tsx` 中使用 `@coze/web-sdk/react` 的 `CozeWebSDK` 容器。
- `src/services/cozeApi.ts` 中保留了直接请求 Coze `/v3/chat` 的 SSE 封装。

当前没有重构 Token 逻辑。需要注意：项目中存在硬编码 Token。如果仓库公开，Token 有泄露风险。正式上线建议改为：

- 重新生成 Token。
- 将 Token 放到服务端或环境变量中。
- 避免把高权限 Token 放进前端源码。

## 5. 数据模型

核心类型定义在 `src/types/index.ts`。

### 5.1 HeritageItem

```ts
export interface HeritageItem {
  id: string;
  name: string;
  category: HeritageCategory;
  region: string;
  level?: HeritageLevel;
  inheritor?: string;
  description: string;
  imageUrl?: string;
  images?: string[];
  videoUrl?: string;
  audioUrl?: string;
  details?: string;
  history?: string;
  techniques: string[];
  masters?: string[];
  tags?: string[];
  featured?: boolean;
  isFavorite?: boolean;
}
```

### 5.2 HeritageLevel

```ts
export type HeritageLevel =
  | 'national'
  | 'provincial'
  | 'municipal';
```

等级显示文本在 `src/App.tsx` 的 `HERITAGE_LEVEL_LABELS` 中维护：

- `national`：国家级
- `provincial`：省级
- `municipal`：市级

### 5.3 图片路径

`src/data/heritageData.ts` 使用 `imagePath` 生成图片路径：

```ts
const imagePath = (file: string) => `${import.meta.env.BASE_URL}images/heritage/${file}`;
```

这样可以兼容本地开发的 `/` 和 GitHub Pages 的 `/feiyi-web/` 子路径。

## 6. 图片资源规范

图片统一放在：

```text
public/images/heritage/
```

命名建议：

- 使用英文 slug。
- 使用小写字母、数字、短横线。
- 示例：`ergui-shuaijiao.jpg`、`fengning-jianzhi.jpg`、`kuancheng-beigan.jpg`。

新增图片后，需要在 `src/data/heritageData.ts` 中引用：

```ts
imageUrl: imagePath('example-name.jpg'),
images: [imagePath('example-name.jpg')],
```

不要直接使用远程热链。远程图片可能失效，也可能跨域或加载慢。

## 7. 本地存储

`src/services/storage.ts` 负责 localStorage 数据：

| Key | 用途 |
| --- | --- |
| `feiyi_favorites` | 收藏项目 |
| `feiyi_chat_history` | 聊天历史 |
| `feiyi_settings` | 本地设置 |

聊天历史最多保留最近 50 条会话。

存储读取时做了类型校验，避免本地旧数据或异常数据导致页面崩溃。

## 10. Vite base 配置

`vite.config.ts` 中根据 GitHub Actions 环境自动设置 base：

```ts
const repositoryName = process.env.GITHUB_REPOSITORY?.split('/')[1]
const base = process.env.GITHUB_ACTIONS === 'true' && repositoryName
  ? `/${repositoryName}/`
  : '/'
```

作用：

- 本地开发时使用 `/`。
- GitHub Pages 构建时使用 `/feiyi-web/`。
- 保证 JS、CSS、图片资源在 Pages 子路径下正常加载。

- 搜索结果、卡片、详情页在小屏下避免横向溢出。

## 12. 已知注意事项

1. PowerShell 启动时可能出现：

   ```text
   Invoke-Expression: Missing argument in parameter list
   ```

   这是本机 PowerShell 配置噪声，不是项目源码错误。

2. 生产构建可能出现 Vite chunk size warning：

   ```text
   Some chunks are larger than 500 kB after minification
   ```

   原因是项目包含 Three.js、Drei、Coze SDK、动画库等较重依赖。当前不影响部署。后续可考虑动态导入 3D 星球或聊天 SDK。

3. Coze Token 当前在前端源码中，公开仓库存在泄露风险。正式发布建议迁移到后端服务或环境变量方案。

4. 新增非遗项目时，要同时维护：

   - `src/data/heritageData.ts`
   - `public/images/heritage/`
   - 必要时更新知识库 Markdown。

5. `dist/` 是构建产物，通常不需要手动修改。

## 13. 常见维护任务

### 13.1 新增非遗项目

1. 把图片放入 `public/images/heritage/`。
2. 在 `src/data/heritageData.ts` 新增项目对象。
3. 确保 `id` 唯一。
4. 确保 `category` 属于 `HeritageCategory`。
5. 确保 `techniques` 至少是空数组或有效数组。
6. 运行检查：

```powershell
npx tsc -p tsconfig.app.json --noEmit --pretty false
npm run lint
npm run build
```

### 13.2 调整首页星球转速

修改 `src/App.tsx`：

```tsx
autoRotateSpeed={0.3}
```

数值越大旋转越快。建议范围：

- `0.15`：很慢
- `0.3`：当前速度
- `0.45`：明显更活跃
- `0.6` 以上可能会显得太急

### 13.3 调整首页滑动提示显示时间

修改 `src/index.css`：

```css
animation: scrollHintLifecycle 5.2s cubic-bezier(0.2, 0.75, 0.22, 1) 0.65s forwards;
```

含义：

- `5.2s`：完整生命周期。
- `0.65s`：延迟出现。
- `forwards`：动画结束后保持隐藏状态。

### 13.4 更新 GitHub 上的版本

```powershell
git add .
git commit -m "更新非遗网站"
git push origin main
```

然后到 GitHub 的“操作”页面等待部署成功。

## 15. 后续优化建议

- 将 Coze Token 从前端源码中移除。
- 对 3D 星球和 Coze SDK 做动态导入，降低首屏 JS 体积。
- 为百科列表增加分页或虚拟滚动。
- 为图片增加压缩和统一尺寸规范。
- 增加自动化视觉回归测试。
- 增加数据校验脚本，避免新增项目字段缺失。
- 将非遗知识库拆分为结构化 JSON 或 CMS 数据源。
